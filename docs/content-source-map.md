## 1) Document Purpose and Implementation Use

This document defines the authoritative content scope for each public-facing page of the static-first epoxy and concrete coatings website. It exists to prevent content drift during future implementation, copy updates, and design iteration.

### Primary Objectives
- Establish what each page must accomplish in the buyer journey.
- Specify which messages belong in which sections.
- Define what content must not be placed on each page.
- Maintain consistent CTA balance between phone calls and estimate submissions.
- Preserve business positioning: mid-market, quality-first, local residential specialist.

### Intended Users
- Developers implementing page templates and components.
- Designers structuring page layouts and section hierarchy.
- Content editors writing or revising production copy.
- Future Codex runs generating production-ready content.

## 2) Global Content Control Rules

### 2.1 Global Messaging Rules
- Lead with homeowner outcomes: durability, safety, curb appeal, cleanability, and long-term value.
- Reinforce local trust: Charleston, SC and surrounding communities.
- Keep positioning consistent: professional, consultative, quality-first, premium-value (not bargain-basement).
- Mention primary service focus repeatedly and clearly: **garage coatings, porch coatings, pool deck coatings**.
- Confirm residential focus; avoid drifting into broad commercial messaging unless explicitly added later as a strategic expansion.

### 2.2 Tone Rules
- Use consultative, professional, clear language.
- Avoid hype, aggressive claims, or “lowest-price” framing.
- Use confidence without overpromising.
- Favor plain-language explanations over technical jargon.
- When technical detail is needed, keep it concise and tie it to homeowner benefit.

### 2.3 CTA Balance Rules
- Maintain **equal emphasis** between call and estimate request across high-intent pages.
- Required recurring CTAs:
  - **Call Now** (or equivalent direct-call CTA)
  - **Get Estimate** (or equivalent form CTA)
- Never hide phone-first users behind long form-only flows.
- Never make estimate request the only conversion path.

### 2.4 Local Relevance Rules
- Reference Charleston, SC and nearby service communities naturally.
- Standard service area: within 40 miles of ZIP 29445.
- Include customer-facing note where relevant: outside standard area may include a small travel fee.
- Avoid broad “statewide” or “nationwide” language.

### 2.5 Content Hierarchy Rules
- Structure each page from most decision-critical content to supporting detail.
- Above the fold: value proposition + trust + dual CTA.
- Mid page: proof and process clarity.
- Lower page: FAQs/supporting objections + CTA reinforcement.
- Do not bury conversion actions below long technical sections.

### 2.6 Outcome-First vs Technical Content Rules
- Outcome-first content should dominate: appearance transformation, surface longevity, lower maintenance burden, safer traction.
- Technical content is allowed when it supports confidence (e.g., prep quality, coating system durability), but must remain secondary.
- Estimate page may include slightly more structured technical inputs because it is decision and qualification oriented.

## Home Page

4. Process snapshot (simple step sequence)
5. Local credibility strip (Charleston-focused service note)

7. Bottom conversion band with dual CTA

- Long-lasting, professional results.
- Local specialists serving Charleston-area homeowners.
- Premium-value quality over cheapest-price framing.

### What Content Should Appear
- Clear outcome-based headline.
- Service triad summary.
- Credibility markers (professionalism, process discipline, responsiveness).
- Equal visibility call + estimate buttons.

### What Content Should NOT Appear
- Deep technical system breakdowns.
- Full gallery dump (use teaser only).
- Long-form brand history.

### CTA Priorities
1. Get Estimate
2. Call Now
(Displayed with equal visual prominence.)

## Estimate Page

### Page Purpose
Capture qualified leads and provide structured, professional price-range guidance.

### Target Visitor Mindset
“I want realistic pricing guidance and to know my project is a fit.”

### Primary Conversion Goal
Estimate form submission.

### Secondary Conversion Goal
Call for immediate consultation.

### Required Sections
1. Estimate intro and expectation setting (price range, not fixed quote)
2. Form fields (required):
   - Project type
   - Size (preset options and custom square footage)

   - Surface condition
   - Finish type
   - Timeline
   - Zip code
   - Name
   - Phone
   - Email
3. Optional file upload (project photo)
4. Price-range output explanation
5. Minimum project threshold message policy ($1,500 minimum)
6. Service-area note (40-mile radius from 29445; small travel fee possible beyond)
7. Parallel call CTA for high-intent users

### Key Messaging Themes
- Transparent, consultative pricing guidance.
- Professional qualification, not automated hard rejection.
- Easy dual path: form or phone call.

### What Content Should Appear
- Clear statement that results are estimate ranges only.
- Professional message when estimate falls below threshold.
- Optional photo upload prompt to improve estimate quality.

### What Content Should NOT Appear
- Guaranteed/fixed quote language.
- Hard disqualification tone for smaller projects.
- Missing required qualification fields.

### CTA Priorities
1. Submit Estimate Request
2. Call Now

## 4) Cross-Page Content Guardrails (Anti-Drift Checklist)

Use this checklist before publishing or approving copy changes:

- Is messaging outcome-first before technical detail?
- Is Charleston-area relevance present and accurate?
- Are garage/porch/pool deck services represented consistently?
- Are both call and estimate CTAs visible and balanced where appropriate?
- Is estimate language range-based (never fixed quote)?
- Is the $1,500 minimum handled professionally (not as hard rejection)?
- Is the service area boundary and travel fee note consistent?
- Is tone consultative and professional (not hype or discount-first)?
- Does page content align with its defined purpose in this document?
